## ADDED Requirements

### Requirement: Tile type carries biome metadata
The system SHALL define a `Tile` struct with fields `Biome Biome`, `Char rune`, `Color string`, `Elevation float64`, `Moisture float64`. `Biome` SHALL be an `int` type with named constants for all nine biomes: `DeepOcean`, `ShallowWater`, `Beach`, `Forest`, `Plains`, `DenseForest`, `Desert`, `Mountains`, `Snow`.

#### Scenario: Tile fields are populated after generation
- **WHEN** `TileAt` is called for any world coordinate
- **THEN** the returned `Tile` has a non-zero `Char`, a non-empty `Color`, and an `Elevation` in the range `[0, 1]`

### Requirement: Chunk is 32×32 tiles generated from noise
The system SHALL define a `Chunk` struct with a `[32][32]Tile` field. Each tile SHALL be generated by sampling two `opensimplex-go` noise functions at `scale = 0.07` using the tile's world-space coordinates.

#### Scenario: Same chunk coordinates always yield the same chunk
- **WHEN** `TileAt` is called twice for the same world coordinate with the same global seed
- **THEN** both calls return an identical `Tile`

#### Scenario: Adjacent chunks have no seam artifacts
- **WHEN** tiles at the boundary of two adjacent chunks are compared
- **THEN** their `Elevation` values differ by less than `0.2` (smooth noise, no hard jumps)

### Requirement: Biome thresholds match the brief specification
The system SHALL assign biomes according to the following rules evaluated in order:
1. `e < 0.28` → `DeepOcean` (`≋`, `#1a6fa8`)
2. `e < 0.36` → `ShallowWater` (`≈`, `#2e9ecf`)
3. `e < 0.40` → `Beach` (`·`, `#e8c96a`)
4. `e < 0.50` and `m > 0.55` → `Forest` (`♣`, `#2d7a1f`)
5. `e < 0.50` and `m ≤ 0.55` → `Plains` (`░`, `#5aad3f`)
6. `e < 0.62` and `m > 0.45` → `DenseForest` (`♠`, `#3d6b3a`)
7. `e < 0.62` and `m < 0.35` → `Desert` (`~`, `#c8a46a`)
8. `e < 0.78` → `Mountains` (`▲`, `#8fa89c`)
9. `e ≥ 0.78` → `Snow` (`*`, `#ccd9e0`)

#### Scenario: Tile with elevation 0.20 is classified as DeepOcean
- **WHEN** a tile has elevation `0.20`
- **THEN** its `Biome` is `DeepOcean`, `Char` is `≋`, and `Color` is `#1a6fa8`

#### Scenario: Tile with elevation 0.48 and moisture 0.60 is classified as Forest
- **WHEN** a tile has elevation `0.48` and moisture `0.60`
- **THEN** its `Biome` is `Forest`, `Char` is `♣`, and `Color` is `#2d7a1f`

#### Scenario: Tile with elevation 0.85 is classified as Snow
- **WHEN** a tile has elevation `0.85`
- **THEN** its `Biome` is `Snow`, `Char` is `*`, and `Color` is `#ccd9e0`

### Requirement: TileAt accessor generates and caches chunks
The system SHALL provide `TileAt(worldX, worldY int, m *Model) Tile` that derives the chunk coordinate by integer division of `(worldX, worldY)` by 32, checks `Model.chunks`, generates and stores the chunk if absent, then returns the appropriate tile.

#### Scenario: First call for an uncached chunk generates and caches it
- **WHEN** `TileAt` is called for a coordinate whose chunk is not yet in `Model.chunks`
- **THEN** the chunk is added to `Model.chunks` before the tile is returned

#### Scenario: Second call for the same coordinate uses the cache
- **WHEN** `TileAt` is called twice for the same coordinate
- **THEN** `Model.chunks` contains exactly one entry for that chunk's key after both calls
